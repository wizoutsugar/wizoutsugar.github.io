# Enumeration
Let’s enumerate more to determine if any of these services are either misconfigured or running vulnerable versions.

## 1. FTP Enumeration
Using searchsploit shows that it’s vulnerable to a backdoor command execution and it has a Metasploit module but as I am trying to not use Metasploit as practice for OSCP, I’m not going to use it. A quick google search shows that this version is vulnerable to a backdoor command execution that is triggered by entering a string that contains the characters “:)” as the username which will create a shell bound to the port 6200. More details here. It also has a nmap script. I’ve tried to exploit using both -manual&nmap- but it didn’t work.

# Exploitation 1: Samba
The nmap scan output says the samba version is 3.0.20 which is out of date and has exploits available. There is a Metasploit module available for it. Using google I found a python script to exploit it and gain a root shell.



![Screenshot]({{ site.baseurl }}/images/posts/2020/Lame/5.Samba_root)

# Exploitation 2: Distcc
The nmap scan output says the distcc version is 1 which is out of date and has exploits available. There is a Metasploit module available for it. Using google I found a python script to exploit it and gain a shell.
